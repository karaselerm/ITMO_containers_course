# Сравнение практик Docker

## Плохие практики (bad_Dockerfile)

1. **Неспецифичный базовый образ**: `FROM python` (latest) - нестабильная версия
2. **Запуск от root**: `USER root` - уязвимость безопасности
3. **Установка ненужных пакетов**: git, vim, nano - увеличивают размер образа
4. **Неоптимальный порядок копирования**: копирование app/ до requirements.txt - плохое кэширование слоёв
5. **Отсутствие флага --no-cache-dir**: pip кэширует пакеты, увеличивая размер образа
6. **Отсутствие EXPOSE**: не документирует порт

## Хорошие практики (good_Dockerfile)

1. **Специфичный slim-образ**: `FROM python:3.11-slim` - конкретная версия, меньший размер
2. **Непривилегированный пользователь**: создание и использование `appuser` - безопасность
3. **Оптимальный порядок слоёв**: requirements.txt → установка зависимостей → копирование кода - эффективное кэширование
4. **Флаг --no-cache-dir**: уменьшает размер образа
5. **EXPOSE 8000**: документирует используемый порт
6. **Минималистичный образ**: только необходимые зависимости
